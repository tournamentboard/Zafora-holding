Below are end-to-end requirements you can give the developer for Zafora.

Zafora Website Admin Panel Requirements
Main Goal

Build the admin panel so every visible item on the live Zafora website can be edited, managed, updated, hidden, reordered, replaced, or removed without touching code.

This includes:

All page text
All images
All buttons
All links
Navigation menus
Footer content
Services
Projects
Pipeline items
Team members
Contact info
Forms
SEO
Testimonials
Stats
Banners
Announcements
Legal pages
Page sections
Website settings

The admin panel should be simple enough that a company team member can make changes without needing a developer.

Final Requirement Statement

The Zafora website must be fully CMS-driven. No visible website content should be hard-coded unless approved. Any text, image, button, card, section, page, service, project, pipeline item, form, navigation link, SEO setting, or footer content visible on the live website must be manageable from the admin panel.
